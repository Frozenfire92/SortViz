About Sort Viz

Sort Viz is a tool to help anyone understand and visualize sorting algorithms. Sort Viz excels from the competition by providing the tools to visualize sorting algorithms as well as test and develop your own. By helping the user to visualize the algorithm in multiple ways it will help solidify your understanding. Sort Viz also provides a testing mode to test your skills and how well you actually know the algorithm.

About Sorting Algorithms

Sorting algorithms are everywhere, and to put it simply they sort things. You can sort a dataset in ascending or descending order and it isn't limited to just numbers. For real world examples they are used in airports for the schedule, data visualizations, and even your google search.

Algorithms in Sort Viz

Merge sort is an O(n log n) comparison-based sorting algorithm. Most implementations produce a stable sort, meaning that the implementation preserves the input order of equal elements in the sorted output. It is a divide and conquer algorithm that was invented by John von Neumann in 1945.

Quicksort is popular because it is not difficult to implement, works well for a variety of different kinds of input data, and is substantially faster than any other sorting method in typical applications. It is in-place, requires time proportional to N log N on the average to sort N items, and has an extremely short inner loop. On the average, it has O(n log n) complexity, making quicksort suitable for sorting big data volumes. Quicksort is a divide-and-conquer method for sorting as well but It works by partitioning an array into two parts, then sorting the parts independently. It was invented in 1960 by C. A. R. Hoare, and it has been studied and refined by many people since that time.

Shell sort is a sorting algorithm, devised by Donald Shell in 1959, that is a generalization of insertion sort, which exploits the fact that insertion sort works efficiently on input that is already almost sorted. It improves on insertion sort by allowing the comparison and exchange of elements that are far apart. The last step of Shell sort is a plain insertion sort, but by then, the array of data is guaranteed to be almost sorted. Also, a general analysis of the shell sort tends to fall somewhere between O(n) and O(n2), based on the behavior described above.
